## Test 1: Shannon Entropy Calculation

**Given:** A fair six-sided die produces outcomes with equal probability 1/6
**When:** The skeptical-auditor computes the entropy
**Then:**
- Computes H(X) = log₂(6) ≈ 2.585 bits
- Confirms this is the maximum entropy for a 6-outcome distribution (uniform)
- Notes that a biased die has lower entropy — uncertainty is reduced
- References entropy_property = maximum_at_uniform
- Does NOT confuse entropy (uncertainty) with energy (thermodynamic entropy shares the formula but differs in meaning)

## Test 2: Channel Capacity Assessment

**Given:** A binary symmetric channel with crossover probability p = 0.1
**When:** The department computes the channel capacity
**Then:**
- Computes C = 1 - H_b(0.1) = 1 - (-0.1 log₂0.1 - 0.9 log₂0.9) ≈ 0.531 bits per channel use
- States that reliable communication is possible at rates R < 0.531 bits/use
- References coding_theorem = noisy_channel_theorem
- Notes that Shannon's theorem is an existence proof — it guarantees achievability but does not specify the code
- Produces U (Unknown) belief if asked whether a specific code achieves capacity without further analysis

## Test 3: Compression Bound Violation Detection

**Given:** A compression algorithm claims to losslessly compress all 1 MB files to under 512 KB
**When:** The department evaluates the claim
**Then:**
- Identifies the claim as impossible by counting argument: there are 2^(8×1024×1024) distinct 1 MB files but only 2^(8×512×1024) distinct 512 KB files — pigeonhole principle
- References compression_bound = kraft_inequality and shannon_entropy_lower_bound
- Produces F (False) belief for the universal compression claim
- Correctly distinguishes: some files compress well (low entropy), but not all files can simultaneously shrink
- Does NOT accept marketing claims that violate information-theoretic limits

## Test 4: Kolmogorov Complexity and Incompressibility

**Given:** A string of 1,000,000 random bits generated by a fair coin
**When:** The department characterizes the string's Kolmogorov complexity
**Then:**
- States K(x) ≈ 1,000,000 bits (incompressible — the shortest program is essentially the string itself)
- Distinguishes from a structured string like "0101...01" which has K(x) = O(log n) bits
- Notes K(x) is uncomputable in general — only upper bounds are practically achievable
- References incompressibility ::= "K(x)" ">=" "|x|" "-" constant
- Explains why truly random data appears to resist all compression schemes

## Test 5: Policy Entropy Analysis

**Given:** A Demerzel governance policy that specifies exactly one permitted action for every possible input
**When:** The department analyzes the policy's information content
**Then:**
- Computes policy_entropy ≈ 0 (deterministic policy — H = 0 when all probability mass on one outcome)
- Classifies as degenerate_policy_entropy: low entropy = redundant / over-specified
- Notes that zero-entropy policies may be efficient but leave no room for agent judgment or context-sensitivity
- Contrasts with a high-entropy policy that enumerates many options — informative but may be ambiguous
- Does NOT treat low entropy as inherently bad — correct context determines whether rigidity is appropriate

## Test 6: BS Detector Signal-to-Noise Analysis

**Given:** A report containing 10 claims: 3 are falsifiable and cite evidence, 7 are vague hedges ("it may be that," "some argue," "could potentially")

- Computes signal = 3 (falsifiable, precise claims), noise = 7 (vague assertions)
- Computes SNR_bs = 3/7 ≈ 0.43 — below actionability threshold

- References bs_detector_snr ::= "signal" = falsifiable_precise_claims
- Does NOT dismiss the 3 valid claims — separates signal from noise rather than rejecting the whole

## Test 7: Mutual Information and Feature Selection

**Given:** A machine learning pipeline needs to select 3 features from 20 candidates to predict a binary outcome

**Then:**
- Recommends maximizing I(X_i; Y) — mutual information between each feature and the target

- References mi_applications = feature_selection and mi_estimation = knn_estimator

- Produces U (Unknown) belief if MI estimates are based on fewer than ~30 samples per variable
