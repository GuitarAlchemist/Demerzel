# Behavioral Tests: Shake-MetaFix Loop

**Pipeline:** `pipelines/shake-metafix-loop.ixql`
**Version:** 1.0.0
**Date:** 2026-03-23

These tests verify the shake-metafix loop correctly injects chaos poison, runs detection,
classifies gaps, applies metafixes with appropriate governance gates, and measures resilience
improvement. Tests cover the closed-loop cycle from injection through re-measurement.

---

## TC-SMF-01 — Detection scoring correctly distinguishes automated vs spec-only

**Scenario:** The loop runs all detectors against chaos-test.ixql. Detection results must correctly classify each injection's status as automated, spec-only, manual, or delayed.

**Input:**
- `pipelines/chaos-test.ixql` (11 injections)
- F# parser with `analyzeLolli()` implementing L2/L3/L4/teach detection
- `cross-pipeline-deps.ixql` (specification only, no runtime)

**Expected behavior:**
- Injections 1-5 and 11 classified as `status: "automated"` (F# parser catches them)
- Injections 6-7 classified as `status: "spec-only"` (cross-pipeline spec exists, no runtime)
- Injections 8-9 classified as `status: "policy-enforceable"` (concrete rules exist, human-executable)
- Injection 10 classified as `status: "delayed-14d"` (staleness detection has 14-day window)
- R score computed as caught/total with documented counting methodology
- No injection falsely classified as "automated" when only spec-level coverage exists

**Governance gate:** Article 1 (Truthfulness) — detection status must be honest.

## TC-SMF-02 — Metafix proposals respect escalation thresholds

**Scenario:** The loop generates metafix proposals for each gap. Proposals with confidence >= 0.9 are auto-applied; proposals with confidence < 0.9 are escalated for human review.

**Input:**
- 3 remaining gaps (L0 cross-pipeline, L1 cross-pipeline, schema staleness)
- Metafix proposals generated by tars.analyze()

**Expected behavior:**
- Each gap produces a metafix proposal with a confidence score
- Proposals with confidence >= 0.9:
  - Pass `reversibility_check` before application (Article 3)
  - Are applied automatically
  - Generate a rollback path
- Proposals with confidence < 0.9:
  - Are NOT applied automatically
  - Are escalated with reason "confidence below 0.9"
  - Trigger a Discord alert listing the escalated fixes
- No fix is applied without passing the reversibility check

**Governance gate:** Article 6 (Escalation) for low-confidence fixes. Article 3 (Reversibility) for all auto-fixes. Article 9 (Bounded Autonomy) for fix scope.

## TC-SMF-03 — R score re-measurement detects improvement

**Scenario:** After auto-fixes are applied, the loop re-runs all detectors and computes a new R score. If R improved, the improvement patterns are compounded via teach/harvest.

**Input:**
- Pre-fix R score from Step 3
- Auto-fixes applied in Step 5
- Post-fix detection results from Step 7

**Expected behavior:**
- If R improved (post > pre):
  - Improvement patterns are harvested
  - Fix patterns promoted if T >= 0.9
  - `resilience_improvement` taught to seldon
  - Score delta logged to `state/evolution/`
- If R did NOT improve (post <= pre):
  - Discord alert sent: "R score did NOT improve after auto-fixes. Human review needed."
  - No compound/teach/promote occurs
  - Escalation is triggered (Article 6)

**Governance gate:** Article 6 (Escalation) on failed improvement. Compound block only fires on success.

## TC-SMF-04 — History record preserves full audit trail

**Scenario:** After the cycle completes, `state/resilience/history.json` is updated with a new record containing all cycle data.

**Input:**
- Completed shake-metafix cycle with scoring, gaps, fixes, and re-measurement

**Expected behavior:**
- New record appended to `records` array (not replacing previous records)
- Record contains:
  - `cycle_id` with incrementing identifier
  - `timestamp` in ISO 8601 format
  - `overall_score` as a float between 0.0 and 1.0
  - `injections_total` and `injections_caught` counts
  - `level_scores` for each level (L0 through schema)
  - `gaps` array with per-injection status
  - `metafixes_applied` array with gap, fix, level_improved, and confidence
  - `level_deltas` showing before/after/delta for each level
  - `trend` as "improving", "stable", or "degrading"
  - `note` with human-readable summary
- Previous records (chaos-001, chaos-002) are preserved unchanged
- Trend is computed correctly: improving if score > previous, stable if equal, degrading if lower

**Governance gate:** Article 7 (Auditability) — full audit trail, no record deletion.

## TC-SMF-05 — Constitutional compliance metafix catches policy-level poison

**Scenario:** The constitutional-compliance-policy.yaml (created as a metafix) is applied to the chaos-test-policy.yaml. The policy should detect multiple violations.

**Input:**
- `chaos-test-policy.yaml` with:
  - Empty `constitutional_basis: []`
  - Empty `principles: []`
  - BS description: "Leveraging synergistic governance paradigms for holistic alignment optimization"
  - `schema_ref` pointing to non-existent file
  - Rule with no constitutional article reference

**Expected behavior:**
- CC-01 violation: `constitutional_basis` is empty (severity: error)
- CC-02: Not applicable (no entries to validate)
- CC-03 violation: Description BS score 4/4 (specificity 0.05, falsifiability 0.0 — both below thresholds)
- CC-04 violation: `placeholder-rule` has no `article` field (severity: warning)
- CC-05 violation: `schema_ref` points to non-existent file (severity: error)
- CC-06 violation: `principles: []` is empty with no justification (severity: warning)
- Total: 4 violations (2 error, 2 warning)

**Governance gate:** Article 5 (Non-Deception) — detecting misleading policy declarations.

## TC-SMF-06 — Loop handles zero-gap scenario gracefully

**Scenario:** If all 11 injections are caught (future state when L0/L1/schema are automated), the loop should report perfect score and skip metafix generation.

**Input:**
- Hypothetical state: all detectors return findings > 0 for all injections

**Expected behavior:**
- R score = 1.0 (11/11)
- `gaps` array is empty
- Steps 4-6 (metafix generation, auto-apply, escalation) are skipped
- Steps 7-8 (re-measurement) run but produce same score (no fixes to measure)
- Step 9 fires (R is not degraded) but with no new patterns to compound
- Step 10 does NOT fire (R did not fail to improve — it was already perfect)
- History record shows `trend: "stable"` (1.0 -> 1.0)
- Discord alert reports: "R=1.0, Gaps: 0, Auto-fixed: 0, Escalated: 0"
- No artifacts created (zero-gap means zero metafixes — anti-LOLLI compliant)

**Governance gate:** Article 4 (Proportionality) — no action when no action is needed.
