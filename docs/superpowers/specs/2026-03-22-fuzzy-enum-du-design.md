# Fuzzy Enums & Discriminated Unions — Design Specification

**Date:** 2026-03-22
**Status:** Approved (brainstorm session)
**Scope:** Demerzel (schemas, grammar, skill) + tars (F# types, CE, type provider, MCP tool)

## Overview

Generalize fuzzy membership from tetravalent logic into a universal type system. A `FuzzyEnum<'T>` distributes membership degrees across enum variants. A `FuzzyDU<'T, 'Payload>` adds typed payloads per variant. A computation expression propagates membership through chained operations with configurable strategies.

This unifies 7+ existing ad-hoc fuzzy systems (beliefs, grammar weights, risk levels, persona selection, BS scores, governance decisions, task priorities) under one type with one CE and one set of operations.

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Source of truth | Bidirectional (A+B+C) | Factory of factories — schemas, F# types, and runtime discovery all produce and consume |
| Scope of fuzzy | Full fuzzy DU with CE (C) | Membership + payloads + operations + computation expression |
| Pattern matching | Weighted + threshold (C) | All branches execute by default; `when T(threshold)` gates opt-in |
| Propagation | Configurable strategy (C) | Multiplicative (default), Zadeh, Bayesian, Custom |
| Auto-production | Type provider + watcher + runtime (C) | Compile-time safety, hot-reload, runtime discovery loop |

## Section 1: Core Types

**F# types** (auto-generated by type provider):

```fsharp
/// A probability distribution over enum variants
type FuzzyEnum<'T when 'T : comparison> = {
    Memberships: Map<'T, float>
} with
    member this.ArgMax() = this.Memberships |> Map.toList |> List.maxBy snd |> fst
    member this.ArgMaxValue() = this.Memberships |> Map.toList |> List.maxBy snd |> snd
    member this.IsSharp(threshold) = this.ArgMaxValue() > threshold
    member this.Get(variant: 'T) = this.Memberships |> Map.tryFind variant |> Option.defaultValue 0.0
    static member Pure(variant: 'T, variants: 'T list) =
        { Memberships = variants |> List.map (fun v -> v, if v = variant then 1.0 else 0.0) |> Map.ofList }
    static member Uniform(variants: 'T list) =
        let w = 1.0 / float variants.Length
        { Memberships = variants |> List.map (fun v -> v, w) |> Map.ofList }
```

### Operations

```fsharp
module FuzzyOps =
    /// Renormalize so memberships sum to 1.0
    let renormalize (fe: FuzzyEnum<'T>) : FuzzyEnum<'T> =
        let total = fe.Memberships |> Map.toList |> List.sumBy snd
        if total = 0.0 then
            // All-zero is invalid per fuzzy-membership.md — fall back to uniform
            FuzzyEnum.Uniform(fe.Memberships |> Map.keys |> Seq.toList)
        else { Memberships = fe.Memberships |> Map.map (fun _ v -> v / total) }

    /// Fuzzy AND: min per variant, renormalize
    let AND (a: FuzzyEnum<'T>) (b: FuzzyEnum<'T>) : FuzzyEnum<'T> =
        let keys = Set.union (a.Memberships |> Map.keys |> Set.ofSeq) (b.Memberships |> Map.keys |> Set.ofSeq)
        { Memberships = keys |> Set.toList |> List.map (fun k ->
            k, min (a.Get k) (b.Get k)) |> Map.ofList }
        |> renormalize

    /// Fuzzy OR: max per variant, renormalize
    let OR (a: FuzzyEnum<'T>) (b: FuzzyEnum<'T>) : FuzzyEnum<'T> =
        let keys = Set.union (a.Memberships |> Map.keys |> Set.ofSeq) (b.Memberships |> Map.keys |> Set.ofSeq)
        { Memberships = keys |> Set.toList |> List.map (fun k ->
            k, max (a.Get k) (b.Get k)) |> Map.ofList }
        |> renormalize

    /// Fuzzy NOT: generic inversion. For non-tetravalent enums, inverts and renormalizes.
    /// IMPORTANT: FuzzyEnum<TruthValue> uses specialized NOT that swaps T<->F, preserves U/C
    /// (per existing logic/fuzzy-membership.md). Override via NotStrategy parameter.
    let NOT (a: FuzzyEnum<'T>) : FuzzyEnum<'T> =
        { Memberships = a.Memberships |> Map.map (fun _ v -> 1.0 - v) }
        |> renormalize

    /// Tetravalent-specific NOT: swap T<->F, preserve U and C (backward compatible)
    let NOT_tetravalent (a: FuzzyEnum<TruthValue>) : FuzzyEnum<TruthValue> =
        { Memberships = Map [
            T, a.Get F;
            F, a.Get T;
            U, a.Get U;
            C, a.Get C
          ] }

    /// Sharpen: if argmax exceeds threshold, collapse to discrete
    let sharpen (threshold: float) (fe: FuzzyEnum<'T>) =
        if fe.ArgMaxValue() > threshold then Some (fe.ArgMax())
        else None

    /// Check escalation: does contradictory membership exceed threshold?
    let shouldEscalate (threshold: float) (fe: FuzzyEnum<TruthValue>) =
        fe.Get C > threshold
```

## Section 2: Fuzzy Computation Expression

### Propagation Strategies

```fsharp
type PropagationStrategy =
    | Multiplicative   // multiply memberships, renormalize after each step
    | Zadeh            // min-propagation (classic fuzzy logic)
    | Bayesian         // posterior update: treat downstream as evidence
    | Custom of (float -> float -> float)  // user-defined combiner

module Propagation =
    let combine strategy (a: float) (b: float) =
        match strategy with
        | Multiplicative -> a * b
        | Zadeh -> min a b
        | Bayesian ->
            let num = a * b
            let den = num + ((1.0 - a) * (1.0 - b))
            if den = 0.0 then 0.5  // guard: a=1,b=0 or a=0,b=1 → inconclusive
            else num / den
        | Custom f -> f a b
```

### FuzzyResult

The output type that carries membership weights through computation:

```fsharp
type FuzzyResult<'T> = {
    Outcomes: Map<'T, float>    // result value → combined membership weight
    Trace: FuzzyTrace list      // provenance chain
}

type FuzzyTrace = {
    Step: string                // description of what produced this
    InputMemberships: Map<string, float>
    OutputMemberships: Map<string, float>
    Strategy: PropagationStrategy
}

module FuzzyResult =
    let pure value = { Outcomes = Map [value, 1.0]; Trace = [] }

    let scaleBy weight strategy result =
        { result with Outcomes = result.Outcomes |> Map.map (fun _ v -> Propagation.combine strategy weight v) }

    let merge strategy (results: FuzzyResult<'T> list) =
        let combined =
            results
            |> List.collect (fun r -> r.Outcomes |> Map.toList)
            |> List.groupBy fst
            |> List.map (fun (k, vs) -> k, vs |> List.sumBy snd)
            |> Map.ofList
        let total = combined |> Map.toList |> List.sumBy snd
        let normalized = if total > 0.0 then combined |> Map.map (fun _ v -> v / total) else combined
        { Outcomes = normalized; Trace = results |> List.collect (fun r -> r.Trace) }
```

### FuzzyBuilder

```fsharp
type FuzzyBuilder(strategy: PropagationStrategy) =

    member _.Return(value: 'T) : FuzzyResult<'T> =
        FuzzyResult.pure value

    member _.ReturnFrom(result: FuzzyResult<'T>) : FuzzyResult<'T> =
        result

    member _.Bind(source: FuzzyEnum<'T>, f: 'T -> FuzzyResult<'U>) : FuzzyResult<'U> =
        source.Memberships
        |> Map.toList
        |> List.filter (fun (_, weight) -> weight > 0.001)  // skip negligible
        |> List.map (fun (variant, weight) ->
            let result = f variant
            FuzzyResult.scaleBy weight strategy result)
        |> FuzzyResult.merge strategy

    member _.Zero() : FuzzyResult<'T> =
        { Outcomes = Map.empty; Trace = [] }

    member _.Combine(a: FuzzyResult<'T>, b: FuzzyResult<'T>) : FuzzyResult<'T> =
        FuzzyResult.merge strategy [a; b]

    member _.Delay(f: unit -> FuzzyResult<'T>) = f

    member _.Run(f: unit -> FuzzyResult<'T>) = f()

/// Convenience instances
let fuzzy = FuzzyBuilder(Multiplicative)
let fuzzyZadeh = FuzzyBuilder(Zadeh)
let fuzzyBayes = FuzzyBuilder(Bayesian)
```

## Section 5: BS Decoder — Skill + MCP Tool

### Demerzel Skill

`/demerzel bs-decode [text or URL]`

Skill at `.claude/skills/demerzel-bs-decode/SKILL.md`.

**Pipeline:**

```
1. INPUT text (or fetch URL content)
       ↓
2. STRUCTURAL ANALYSIS
   - specificity_test → FuzzyEnum<Specificity>  {Specific, Generic}
   - falsifiability_test → FuzzyEnum<Falsifiability>  {Testable, Unfalsifiable}
   - commitment_test → FuzzyEnum<Commitment>  {Committed, Vague}
       ↓
3. CONTENT ANALYSIS
   - density_test → FuzzyEnum<Density>  {Substantive, Fluff}
   - factual_density → float (facts per sentence)
   - actionability_test → FuzzyEnum<Actionability>  {Actionable, Inactionable}
       ↓
4. RHETORICAL ANALYSIS
   - device_scan → list of matched rhetorical devices from gov-bs-generators.ebnf
   - evasion_scan → list of matched evasion tactics
   - rhetorical_density → float (devices per sentence)
       ↓
5. SCORING
   - Weighted combination → FuzzyEnum<BsLevel>  {Clear, Mild, Moderate, Serious, Pure}
   - Weights: specificity 0.2, falsifiability 0.15, commitment 0.2,
              density 0.15, factual 0.15, actionability 0.15
       ↓
6. TETRAVALENT MAPPING
   - Clear/Mild → T (real content)
   - Moderate → U (substance unclear)
   - Serious/Pure → C (contradictory — sounds meaningful, says nothing)
       ↓
7. REWRITE SUGGESTION
   - If score > Moderate: generate clear-speech alternative
   - Apply antidote pattern: claim + evidence + specifics + commitment
       ↓
8. OUTPUT
   - BS score (FuzzyEnum<BsLevel>)
   - Per-test scores
   - Devices found
   - Tetravalent mapping
   - Suggested rewrite (if BS detected)
```

## Mathematical Foundation

### Relationship to existing theory

- **Fuzzy set theory (Zadeh 1965):** FuzzyEnum is a finite fuzzy set. Our AND/OR/NOT follow standard fuzzy operations.
- **Probability theory:** FuzzyEnum with sum_to_one constraint is a probability distribution. Bayesian propagation treats it as such.
- **Category theory:** FuzzyEnum is monad-like. Bind = Kleisli composition over fuzzy distributions. Return = Dirac delta (pure/sharp value). Note: the 0.001 epsilon filter in Bind and renormalization in merge mean strict monad laws hold only approximately. This is intentional — practical fuzzy computation requires epsilon-trimming to avoid exponential branch explosion. Left/right identity hold within epsilon; associativity holds within renormalization tolerance.
- **K-theory (Grothendieck):** Governance configurations as bundles. FuzzyEnum over governance variants classifies configurations up to equivalence. Future work.
- **Topos theory:** Tetravalent {T,F,U,C} is a subobject classifier in a Grothendieck topos. FuzzyEnum generalizes this to continuous subobject classification.

### Key invariants

1. Memberships sum to 1.0 (±0.01 tolerance) under `sum_to_one` constraint
2. `FuzzyEnum.Pure(x).ArgMax() = x` (sharp values are fixed points)
3. `AND(a, NOT(a))` does NOT equal zero (fuzzy logic differs from classical)
4. `renormalize` is idempotent
5. CE propagation preserves sum-to-one after each step
6. Backward compatible: discrete enums embed as sharp fuzzy enums

## Governance

- Article 1 (Truthfulness) — fuzzy membership is more truthful than forced discrete classification
- Article 2 (Transparency) — membership degrees and traces expose reasoning
- Article 6 (Escalation) — C > 0.3 triggers escalation (preserved from tetravalent)
- Article 3 (Reversibility) — fuzzy-to-discrete sharpening is irreversible collapse; use only when necessary
- Article 7 (Auditability) — FuzzyResult.Trace provides full provenance chain
- Article 8 (Observability) — membership degrees and traces expose reasoning
- Article 9 (Bounded Autonomy) — runtime discovery (Layer 3) proposes but does not auto-merge schema changes
- Grammar evolution policy — fuzzy enum definitions evolve via distillation
- Confidence thresholds — default sharpening at argmax > 0.8. Maps to governance tiers: >= 0.9 proceed, >= 0.7 note, >= 0.5 confirm, >= 0.3 escalate, < 0.3 do not act
