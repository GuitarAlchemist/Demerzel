# PRD: ga -- Guitar Alchemist Music Theory Platform

**Version:** 1.0 | **Last Updated:** 2026-04-03 | **Status:** Active

---

## Executive Summary

GA is a .NET 10 monorepo combining C# 14 and F# to deliver a music theory platform with REST/GraphQL API, RAG-powered chatbot, React frontend, and a five-layer domain architecture. It covers notes, intervals, scales, chords, fretboard geometry, voice leading, spectral analysis, and AI-powered semantic search using a custom 216-dimension OPTIC-K embedding schema. Orchestrated via .NET Aspire with MongoDB, Jaeger tracing, and Playwright E2E tests.

## Problem Statement

Music theory is fragmented across notation systems, instruments, and pedagogical approaches. Guitarists need a unified platform that connects theory (scales, modes, chord progressions) to practice (fretboard positions, voice leading) to exploration (AI-powered similarity search, chatbot Q&A). GA bridges these domains with a computationally rigorous backend and visual frontend.

## Goals & Success Metrics

### P0 (Must-Have)
- **Build clean**: `dotnet build AllProjects.slnx` + `dotnet test` pass
- **API operational**: GaApi serves REST + GraphQL + SignalR on port 7001
- **Chatbot functional**: Ollama-powered RAG chatbot answers music theory questions

### P1 (Should-Have)
- **OPTIC-K search**: 216-dim musical similarity search across vector backends (InMemory, Qdrant, MongoDB, CUDA)
- **DSL parsing**: VexTab, GuitarPro, MIDI, MusicXML, AsciiTab via F# FParsec parsers
- **Aspire orchestration**: Full dev stack (MongoDB, API, chatbot, dashboards) via single script

### P2 (Nice-to-Have)
- **Music Theory LSP**: Language Server Protocol for the music theory DSL
- **BSP geometry**: Binary Space Partitioning for fretboard visualization
- **Spectral RAG**: Topological analysis of musical structures

## Architecture

```
Five-Layer Dependency Model (strict bottom-up):
  Core → Domain → Analysis → AI/ML → Orchestration

Application Stack:
  .NET Aspire (AppHost)
    ├── GaApi (ASP.NET Core + SignalR + HotChocolate GraphQL)
    ├── GaChatbot (Ollama RAG)
    ├── MongoDB (data store)
    ├── ga-client (React frontend, consumes ga-react-components)
    └── Monitoring: Aspire dashboard :15001, Jaeger :16686, Mongo Express :8081

Coding Standards:
  C# 14: Records, primary constructors, collection expressions, ROP (Result<T>)
  F#: FParsec for DSL, PascalCase YAML, CLIMutable records
  Frontend: React 18 strict TS, MUI 5, R3F for 3D
  Zero warnings policy, pre-commit format hooks
```
